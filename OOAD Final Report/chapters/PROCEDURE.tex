\section{Functional Requirements}

\subsection{Vendor Subscription, Registration and Login}
This system handles the complete onboarding process for vendors joining the platform. It includes a registration form where vendors provide business details and upload legal documents (Trade License, E-TIN). Upon successful registration, vendors select and pay for a subscription plan (Basic or Professional) through integrated payment gateways. The login system implements Two-Factor Authentication (2FA) using OTP sent to email or phone, ensuring secure access to vendor dashboards. Each vendor receives a unique tenant ID and subdomain for their isolated store.

\begin{itemize}
    \item \textbf{FR-1.1:} The system shall allow vendors to register by submitting business details and uploading verification documents (Trade License, E-TIN).
    \item \textbf{FR-1.2:} The system shall allow vendors to select and process payment for a subscription plan (Basic or Professional).
    \item \textbf{FR-1.3:} The system shall enforce Two-Factor Authentication (2FA) via email or SMS OTP during vendor login.
    \item \textbf{FR-1.4:} The system shall automatically provision a unique tenant database and subdomain for the vendor upon successful registration and payment. \textit{[Maps to UC: Vendor Registration]}
\end{itemize}

\subsection{Discount, Offers and Theme Customization}
This system enables vendors to create promotional campaigns and personalize their storefront appearance. Vendors can create percentage-based discounts (e.g., 20\% off), fixed-amount discounts, and "Buy X Get Y" offers (e.g., Buy 2 Get 1 Free). They can set start/end dates and usage limits (e.g., first 50 customers). The theme customization feature allows vendors to upload logos, select brand colors, choose layout templates, and customize navigation menus—all with live preview before publishing changes to their live store.

\begin{itemize}
    \item \textbf{FR-2.1:} The system shall allow vendors to configure percentage-based, fixed-amount, and conditional ("Buy X Get Y") discount rules.
    \item \textbf{FR-2.2:} The system shall allow vendors to define validity periods (start and end dates) and usage limits for all promotional offers.
    \item \textbf{FR-2.3:} The system shall provide an interface for vendors to upload brand logos, select color palettes, and modify layout templates.
    \item \textbf{FR-2.4:} The system shall generate a live visual preview of theme modifications before they are published to the public storefront. \textit{[Maps to UC: Customize Store]}
\end{itemize}

\subsection{Payment Gateway System}
This system processes customer payments for orders placed on vendor stores. It integrates with multiple payment gateways including bKash, Nagad, Visa, and MasterCard, as well as supporting Cash on Delivery (COD). Customers select their preferred method during checkout. For online payments, the system redirects to the gateway and waits for confirmation before finalizing the order. Payment statuses (Paid, Failed, Pending) are tracked and updated accordingly. The system also generates transaction logs for audit purposes.

\begin{itemize}
    \item \textbf{FR-3.1:} The system shall provide customers with checkout options for bKash, Nagad, Visa, MasterCard, and Cash on Delivery (COD).
    \item \textbf{FR-3.2:} The system shall securely redirect users to third-party payment gateways for online transactions.
    \item \textbf{FR-3.3:} The system shall automatically update the order status to "Paid," "Failed," or "Pending" based on the webhook response from the payment gateway.
    \item \textbf{FR-3.4:} The system shall generate and securely store immutable transaction logs for all payment attempts. \textit{[Maps to UC: Make Payment]}
\end{itemize}

\subsection{Inventory Management and Analytics}
This module allows vendors to manage their product stock efficiently. Vendors can add new products, update stock quantities, set reorder levels, and receive low stock alerts. The system tracks stock movements (sales, restocks, adjustments) and maintains accurate inventory counts. Vendors can view inventory valuation, monitor stock turnover rates, and generate inventory reports. The module ensures that each vendor's inventory data remains isolated within their own tenant database, preventing data leakage between competing businesses.

\begin{itemize}
    \item \textbf{FR-4.1:} The system shall allow vendors to add, modify, and delete product listings within their isolated tenant database.
    \item \textbf{FR-4.2:} The system shall automatically decrement stock quantities upon successful order placement.
    \item \textbf{FR-4.3:} The system shall trigger automated low-stock alerts to the vendor dashboard when inventory falls below predefined reorder levels.
    \item \textbf{FR-4.4:} The system shall generate and display inventory valuation and stock turnover reports on the vendor analytics dashboard. \textit{[Maps to UC: Add Products/Inventory Control]}
\end{itemize}

\subsection{Customer Feedback, Product Ratings and Reviews}
This module enables customers to share their product experiences and helps vendors build trust. After purchasing a product, verified customers can submit star ratings (1 to 5 stars) and written reviews, optionally with photos. Reviews are displayed on product pages along with average ratings. Vendors can view pending reviews, approve or reject inappropriate content, and respond to customer feedback. The system tracks helpful votes and provides review analytics (average rating, distribution, response rates) to help vendors improve their products and customer satisfaction.

\begin{itemize}
    \item \textbf{FR-5.1:} The system shall allow verified purchasers to submit a numerical rating (1 to 5 stars) and a textual review with optional image uploads.
    \item \textbf{FR-5.2:} The system shall dynamically calculate and display the aggregate average rating for each product on the storefront.
    \item \textbf{FR-5.3:} The system shall provide vendors with moderation tools to approve, reject, or reply to customer reviews.
    \item \textbf{FR-5.4:} The system shall aggregate review data to display response rates and rating distributions on the vendor analytics dashboard. \textit{[Maps to UC: Review Products]}
\end{itemize}

\section{Non-Functional Requirements}

\subsection{Data Isolation \& Security (Confidentiality)}
Metric: Since we are using a multi-tenant "Database-per-Tenant" design, the system must ensure 100\% data isolation. No cross-tenant query should be possible.

\subsection{Scalability \& Performance}
Metric: The Analytical Board and Inventory Management systems must handle concurrent updates. The system should maintain a response time of < 2 seconds for dashboard loading, even when 500+ vendors are accessing their respective databases simultaneously.

\subsection{High Availability \& Reliability}
Metric: Given that this is a subscription-based "Independent Storefront" model, the system must guarantee 99.9\% uptime. If the Payment Gateway or the Database-per-Tenant service fails, it must have a failover mechanism so vendors don't lose sales during peak promotional hours.

\section{Use Case Specifications}

\subsection{Vendor Registration and Store Creation}
\textbf{Use Case Name:} Vendor Registration and Store Creation\\
\textbf{Actor}: Vendor (Entrepreneur)\\
\textbf{Description:} Allows entrepreneurs to register their business, subscribe to the platform, and create an independent digital storefront with isolated data storage.\\
\textbf{Preconditions:} Vendor has valid business and contact information. Vendor has internet access.

\vspace{0.2cm}
\textbf{Main Flow:} 
\begin{enumerate}
    \item Vendor enters business registration details and uploads necessary documents (Trade License, E-TIN).
    \item Vendor selects a subscription plan (Basic or Professional).
    \item Vendor completes account verification via email/OTP.
    \item Vendor is redirected to the payment gateway to process the subscription fee.
    \item Upon successful payment, the platform automatically provisions a unique tenant ID and an isolated database.
    \item Platform activates the digital storefront and grants dashboard access.
\end{enumerate}

\textbf{Alternative Flows:}
\begin{itemize}
    \item \textbf{4a. Payment Failure:} If the payment transaction fails or is cancelled, the system saves the registration as "Pending" and prompts the vendor to retry payment. The storefront is not provisioned.
    \item \textbf{1a. Duplicate Email:} If the vendor enters an email already in use, the system prompts them to log in or reset their password.
\end{itemize}

\vspace{0.2cm}
\textbf{Postconditions:} Vendor account is created and verified. Independent storefront is activated with isolated data storage.


\vspace{0.5cm}
\subsection{Store Operations and Inventory Management}
\textbf{Use Case Name:} Store Operations and Inventory Management\\
\textbf{Actor:} Vendor\\
\textbf{Description:} Allows vendors to manage store operations through a centralized dashboard including product management, inventory updates, promotions, analytics, and storefront customization.\\
\textbf{Preconditions:} Vendor account exists. Vendor is logged into the platform and 2FA is verified.

\vspace{0.2cm}
\textbf{Main Flow:} 
\begin{enumerate}
    \item Vendor accesses the centralized dashboard.
    \item Vendor navigates to the Inventory section to add new product listings or update existing stock quantities.
    \item Vendor navigates to the Promotions section to configure percentage-based or fixed discounts.
    \item Vendor views sales analytics and stock turnover reports.
    \item Vendor saves all changes.
    \item System updates the vendor's isolated database and instantly reflects changes on the live storefront.
\end{enumerate}

\textbf{Alternative Flows:}
\begin{itemize}
    \item \textbf{2a. Invalid Product Data:} If the vendor enters negative values for price or stock, the system blocks the update and displays an error message highlighting the invalid fields.
\end{itemize}

\vspace{0.2cm}
\textbf{Postconditions:} Store products, inventory, and promotions are updated. Storefront configuration changes are applied securely within the tenant's database.


\vspace{0.5cm}
\subsection{Checkout, Payment, and Order Tracking}
\textbf{Use Case Name:} Product Purchase and Order Fulfillment\\
\textbf{Actor:} Customer\\
\textbf{Description:} Allows customers to purchase products using multiple payment methods and track order delivery through integrated logistics services.\\
\textbf{Preconditions:} Customer selects products. Products are available in stock.

\vspace{0.2cm}
\textbf{Main Flow:} 
\begin{enumerate}
    \item Customer adds products to the shopping cart and proceeds to checkout.
    \item Customer enters shipping and contact details.
    \item System calculates total price including delivery charges via logistics API integration.
    \item Customer selects a payment method (e.g., bKash, Nagad, Visa, or Cash on Delivery).
    \item System redirects to the respective payment gateway (if online payment is selected).
    \item Upon successful payment, the system confirms the order, decrements the vendor's inventory, and generates an invoice.
    \item System provides the customer with a live delivery tracking ID.
\end{enumerate}

\textbf{Alternative Flows:}
\begin{itemize}
    \item \textbf{4a. Cash on Delivery (COD):} If the customer selects COD, the system bypasses the payment gateway, confirms the order immediately, and sets the payment status to "Pending (COD)".
    \item \textbf{5a. Gateway Timeout/Failure:} If the online payment fails, the system redirects the customer back to the checkout page to select an alternative payment method without clearing their cart.
    \item \textbf{6a. Out of Stock During Checkout:} If another user purchases the last item before this transaction completes, the system halts the checkout, alerts the user, and initiates a refund if payment was captured.
\end{itemize}

\vspace{0.2cm}
\textbf{Postconditions:} Order is recorded securely in the system. Inventory is updated. Customer and vendor can track the delivery status.


\vspace{0.5cm}
\subsection{Customer Ratings and Reviews}
\textbf{Use Case Name:} Customer Ratings and Reviews\\
\textbf{Actor:} Customer\\
\textbf{Description:} Allows verified customers to submit a star rating and written review for purchased products, which vendors can then moderate or respond to.\\
\textbf{Preconditions:} The customer has successfully purchased and received the product.

\vspace{0.2cm}
\textbf{Main Flow:}
\begin{enumerate}
    \item Customer navigates to their order history and selects "Write a Review."
    \item Customer selects a star rating (1-5) and enters textual feedback.
    \item Customer submits the review.
    \item System verifies the purchase record against the database.
    \item System updates the product's average rating and displays the review on the public storefront.
\end{enumerate}

\textbf{Alternative Flows:}
\begin{itemize}
    \item \textbf{4a. Unverified Purchase:} If the system cannot verify the purchase history for that user, it prevents submission and displays an error message.
\end{itemize}

\vspace{0.2cm}
\textbf{Postconditions:} The product rating is updated, and the review is visible to other users.


\vspace{0.5cm}
\subsection{Platform Administration and Monitoring}
\textbf{Use Case Name:} Platform Administration and Monitoring\\
\textbf{Actor:} System Admin\\
\textbf{Description:} Allows the platform administrator to monitor overall system health, manage vendor subscriptions, and enforce platform policies.\\
\textbf{Preconditions:} Admin is successfully logged in with high-level administrative privileges.

\vspace{0.2cm}
\textbf{Main Flow:}
\begin{enumerate}
    \item Admin logs into the central administrative dashboard.
    \item Admin views platform-wide metrics (total active tenants, revenue, server health).
    \item Admin selects a specific vendor account to review their subscription status.
    \item Admin issues a suspension or warning if the vendor violates platform policies.
    \item System restricts the vendor's access and disables their storefront.
\end{enumerate}

\textbf{Alternative Flows:}
\begin{itemize}
    \item \textbf{4a. Subscription Renewal Override:} Admin manually extends a vendor's subscription if there was a billing error on the platform's end.
\end{itemize}

\vspace{0.2cm}
\textbf{Postconditions:} Platform policies are enforced, and overall system health is maintained.
