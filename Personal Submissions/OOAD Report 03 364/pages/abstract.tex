This lab activity focuses on understanding and writing user stories as part of the software requirements specification. User stories were used to describe system functionalities from the perspective of different users in a simple and understandable way. Through this lab, several user stories were identified for a Multi-Tenant E-Commerce System, helping to translate user needs into clear functional requirements. The activity emphasizes how user stories improve communication between stakeholders and support user-centered system design.
