\chapter{CONCLUSIONS}\label{ch:conclusion}

This lab activity successfully demonstrated the use of user stories to capture system requirements from a user’s point of view. By writing user stories for different features of the Multi-Tenant E-Commerce System, the lab helped in understanding how user needs can be expressed clearly and concisely. User stories simplify requirement analysis, reduce misunderstandings, and ensure that system development remains focused on real user expectations. Overall, this activity highlights the importance of user stories in designing user-friendly and effective software systems.
