This lab report presents the use case modeling for a Multi-Tenant E-Commerce System (M-TECS) , a cloud-based SaaS platform designed to assist small business owners who face challenges in acquiring customers through social media. The system provides an integrated solution for vendors to create customized online stores, manage products and offers, and process customer orders, while enabling customers to browse, purchase products, and track orders. The report includes two UML use case diagrams: the first illustrates the core system actors (Admin, Vendor, Customer) and their high-level interactions; the second provides a detailed breakdown of vendor-specific functionalities related to offers and store customization. These diagrams were created using Lucidchart to visually represent system requirements and actor-system relationships. The objectives, methodology, results, and conclusions of this modeling exercise are discussed, highlighting how use case diagrams serve as a foundational tool for system analysis and design in the OOAD lab context.
