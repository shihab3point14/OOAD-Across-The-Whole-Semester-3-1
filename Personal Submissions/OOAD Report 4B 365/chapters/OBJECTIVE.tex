\chapter{OBJECTIVE}\label{obj}

The primary objectives of this lab exercise are as follows:

\begin{enumerate}
    \item \textbf{To Understand System Behavior Using Sequence Diagrams:} Analyze different real-world systems and represent their dynamic behavior using UML sequence diagrams, focusing on the interaction between actors and system components.

    \item \textbf{To Identify Actors, Components, and Message Flow:} Determine the main actors (such as teachers, students, administrators, customers, and employees) and illustrate how messages are exchanged between them and the system in chronological order.

    \item \textbf{To Model System Constraints and Business Rules:} Represent important system constraints such as role-based authorization, prerequisite validation, financial verification, and record archiving within sequence diagrams.

    \item \textbf{To Visualize Complex Workflows:} Design sequence diagrams that capture multi-step processes such as school attendance tracking, university enrollment, online shopping order fulfillment, and examination management.

    \item \textbf{To Develop UML Modeling Skills:} Use a web-based UML modeling tool (sequenceDiagram.org) to create clear and structured sequence diagrams that document system logic effectively.

    \item \textbf{To Support System Analysis and Design:} Produce visual documentation that helps identify potential bottlenecks, validate system requirements, and assist in the early stages of system development.
\end{enumerate}
