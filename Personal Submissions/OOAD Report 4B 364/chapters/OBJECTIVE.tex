\chapter{OBJECTIVE}\label{obj}


The primary objectives of this lab exercise are as follows:

\begin{enumerate}
    \item \textbf{To Identify System Actors and Use Cases:} Determine the primary actors interacting with the Multi-Tenant E-Commerce System (Admin, Vendor, Customer) and the key functionalities (use cases) each actor performs.
    
    \item \textbf{To Model High-Level System Interactions:} Create a comprehensive UML use case diagram that illustrates the overall scope of the system and the relationships between actors and their core activities (registration, store management, ordering, etc.).
    
    \item \textbf{To Elaborate Vendor-Specific Functionalities:} Develop a detailed use case diagram focusing specifically on the vendor's capabilities for creating offers and customizing their storefront, including complex relationships like \texttt{\textless{}\textless{}include\textgreater{}\textgreater{}} and generalizations.
    
    \item \textbf{To Demonstrate UML Modeling Proficiency:} Apply object-oriented analysis and design principles using industry-standard modeling tools (Lucidchart) to visually document system requirements.
    
    \item \textbf{To Establish a Foundation for Detailed Design:} Provide clear, visual documentation that bridges the gap between high-level project scope (as defined in the SRS) and detailed functional requirements for developers and stakeholders.
\end{enumerate}
